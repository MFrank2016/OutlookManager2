# 快速指南：修改用户密码

## 🎯 适用场景

- 用户忘记密码需要重置
- 新用户需要设置初始密码
- 安全策略要求定期更换密码
- 用户账户被锁定需要重置

## ⚡ 快速操作步骤

### 1️⃣ 进入用户管理页面

```
登录管理员账户 → 点击"管理面板" → 选择"用户管理"标签
```

### 2️⃣ 找到目标用户

在用户列表中找到需要修改密码的用户行

### 3️⃣ 点击"改密码"按钮

每个用户行都有三个操作按钮：
- **编辑** - 修改用户信息
- **改密码** - 修改用户密码 ⬅️ 点击这个
- **删除** - 删除用户

### 4️⃣ 填写新密码

在弹出的模态框中：
1. 确认用户名（自动显示，不可修改）
2. 输入新密码（至少6位）
3. 再次输入新密码确认

### 5️⃣ 确认修改

点击"确认修改"按钮，系统会：
- 验证两次密码是否一致
- 验证密码长度是否符合要求
- 更新数据库中的密码
- 显示成功提示

### 6️⃣ 通知用户

修改成功后，通过安全渠道告知用户新密码

## ⚠️ 注意事项

### 密码要求
- ✅ 最少6位字符
- ✅ 两次输入必须一致
- ⚠️ 建议使用强密码（包含大小写字母、数字、特殊字符）

### 安全提示
- 🔒 只有管理员可以修改密码
- 🔒 密码会被加密存储
- 🔒 建议通过安全渠道（加密邮件、即时通讯）告知用户新密码
- 🔒 不要在不安全的渠道（如普通邮件、短信）发送密码

### 操作建议
- 📝 记录密码修改操作（谁、何时、为谁修改）
- 📝 建议用户首次登录后立即修改密码
- 📝 定期提醒用户更换密码

## 🐛 常见问题

### Q1: 为什么我看不到"改密码"按钮？
**A**: 只有管理员账户才能看到并使用此功能。请确认您是以管理员身份登录。

### Q2: 提示"两次输入的密码不一致"怎么办？
**A**: 请仔细检查两次输入的密码是否完全相同，包括大小写。

### Q3: 提示"密码长度至少为6位"怎么办？
**A**: 请输入至少6个字符的密码。建议使用更长、更复杂的密码以提高安全性。

### Q4: 修改密码后用户无法登录？
**A**: 请确认：
1. 密码修改是否成功（有成功提示）
2. 告知用户的密码是否正确
3. 用户输入时是否注意大小写
4. 用户账户是否处于"启用"状态

### Q5: 可以修改管理员自己的密码吗？
**A**: 可以。管理员也可以使用此功能修改自己的密码。

### Q6: 修改密码后原密码还能用吗？
**A**: 不能。密码修改后，原密码立即失效，只能使用新密码登录。

## 🔧 故障排除

### 问题：点击"改密码"按钮没有反应
**解决方案**:
1. 检查浏览器控制台是否有错误
2. 刷新页面重试
3. 清除浏览器缓存
4. 检查网络连接

### 问题：提示"修改密码失败"
**解决方案**:
1. 检查用户是否存在
2. 确认您有管理员权限
3. 检查服务器日志
4. 联系系统管理员

### 问题：模态框无法关闭
**解决方案**:
1. 点击模态框外的灰色区域
2. 点击右上角的 × 按钮
3. 按 ESC 键（如果支持）
4. 刷新页面

## 🎓 最佳实践

### 密码策略建议
```
✅ 推荐密码格式：
   - 至少8位字符
   - 包含大写字母 (A-Z)
   - 包含小写字母 (a-z)
   - 包含数字 (0-9)
   - 包含特殊字符 (!@#$%^&*)

❌ 避免使用：
   - 简单密码（如 123456, password）
   - 用户名或邮箱作为密码
   - 连续字符（如 abcdef, 123456）
   - 常见单词（如 admin, user）
```

### 密码管理建议
1. **定期更换**: 建议每3-6个月更换一次密码
2. **唯一性**: 不要在多个系统使用相同密码
3. **保密性**: 不要与他人分享密码
4. **安全存储**: 使用密码管理器存储密码
5. **首次登录**: 新用户首次登录后应立即修改密码

### 操作记录建议
建议管理员记录以下信息：
- 修改时间
- 操作管理员
- 目标用户
- 修改原因
- 是否已通知用户

示例记录格式：
```
日期: 2025-11-02 11:30
管理员: admin
用户: frank
原因: 用户忘记密码
状态: 已修改并通知用户
```
